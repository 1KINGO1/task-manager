## Опис
Менеджер завдань - консольний застосунок для обліку ппроєктів та завдань, що належать цим проєктам.

- Проєкт містить логічно пов'язані завдання.
- Завдання не може існувати без проєкту та не може належати кільком проєктам одночасно.
- Для сутностей зберігаються дані, а також підтримуються обчислювані поля (прогрес проєкту, простроченість завдання).

---

## Сутності та поля

### Проєкт (Project) - сутність першого рівня
- Унікальний ідентифікатор (`Id`)
- Назва (`Name`)
- Короткий опис (`Description`)
- Тип проєкту (`ProjectType` - enum: Educational, Work, Personal, Research, OpenSource)
- Прогрес (в %) - обчислюване поле (тільки в `ProjectModel`):  
  `CompletedTasks / TotalTasks * 100`

### Завдання (TaskItem) - сутність другого рівня
- Унікальний ідентифікатор (`Id`)
- Ідентифікатор проєкту (`ProjectId`)
- Назва (`Title`)
- Опис / технічні характеристики (`Description`)
- Пріоритет (`TaskPriority` - enum: Critical, High, Medium, Low)
- Дедлайн (`DueDate`)
- Позначка виконання (`IsCompleted`)
- Простроченістьь - обчислюване поле (тільки в `TaskItemModel`):  
  `IsOverdue = !IsCompleted && DueDate < DateTime.Today`

---

## Структура рішення

Рішення складається з 4 проєктів:

### `TaskManager.Data` - бібліотека класів для зберігання
Містить класи, що відповідають виключно за зберігання даних (без обчислюваних полів, без колекцій пов'язаних сутностей):
- `ProjectType` - enum типів проєкту
- `TaskPriority` - enum пріоритетів завдань
- `ProjectData` - дані проєкту (Id, Name, Description, Type)
- `TaskItemData` - дані завдання (Id, ProjectId, Title, Description, Priority, DueDate, IsCompleted)

### `TaskManager.Models` - бібліотека класів для відображення / створення / редагування
Містить класи з обчислюваними полями та методами для роботи з UI:
- `ProjectModel` - модель проєкту з колекцією завдань (`Tasks`), обчислюваним прогресом (`ProgressPercent`), методами `ToShortString()` / `ToDetailedString()` та фабричним методом `FromData()`
- `TaskItemModel` - модель завдання з обчислюваним полем `IsOverdue`, методами `ToShortString()` / `ToDetailedString()` та фабричним методом `FromData()`

### `TaskManager.Services` - бібліотека класів для сервісів
- `FakeStorage` (internal static) - штучне сховище з реалістичними початковими даними (3 проєкти, 12 завдань). Доступ до нього має лише `StorageService`.
- `StorageService` - сервіс для отримання даних зі сховища та їх перетворення у моделі для відображення.

### `TaskManager.ConsoleApp` - консольний застосунок
Точка входу програми. Забезпечує інтерактивну взаємодію з користувачем через консоль.

---

## Розподіл класів Data vs Models

Для кожної сутності створено два класи з різним призначенням (принцип Single Responsibility):

| Клас | Шар | Призначення |
|------|-----|-------------|
| `ProjectData` | Data | Зберігання. Тільки поля даних, без обчислень, без колекції завдань. |
| `ProjectModel` | Models | Відображення / редагування. Додає `Tasks`, `ProgressPercent`, методи форматування. |
| `TaskItemData` | Data | Зберігання. Тільки поля даних, без обчислень. |
| `TaskItemModel` | Models | Відображення / редагування. Додає `IsOverdue`, методи форматування. |

## Логіка консольного застосунку

Після запуску застосунок працює в циклі:

1. **Завантажити та показати список проєктів**
   - Через `StorageService` отримуються проєкти та їх завдання
   - Виводиться коротка інформація: Id, Назва, Тип, Прогрес

2. **Вибрати проєкт** (ввести його номер)
   - Показати деталі проєкту (опис, тип, прогрес)
   - Показати список завдань (назва, пріоритет, дедлайн, статус)

3. **Вибрати завдання** (ввести його номер)
   - Показати повну інформацію (опис, дедлайн, пріоритет, статус, простроченість)

4. **Повернутись до списку проєктів** - ввести `b`

5. **Завершити роботу** - ввести `q`
