## Опис
**Менеджер завдань** - консольний застосунок для обліку **проєктів** та **завдань**, що належать цим проєктам.

- Проєкт містить логічно пов’язані завдання.
- Завдання **не може існувати без проєкту** та **не може належати кільком проєктам одночасно**.
- Для сутностей зберігаються **дані**, а також підтримуються **обчислювані поля** (наприклад, прогрес проєкту, простроченість завдання).

---

## Сутності та поля

### Проєкт (Project)
Зберігає:
- Унікальний ідентифікатор
- Назву
- Короткий опис
- Тип проєкту (**enum**, напр.: Навчальний, Робочий, Особистий)
- **Прогрес (%)** — *обчислюване поле*:  
  `CompletedTasks / TotalTasks * 100`

### Завдання (Task)
Зберігає:
- Унікальний ідентифікатор
- Ідентифікатор проєкту (ProjectId)
- Назву
- Опис / технічні характеристики
- Пріоритет (**enum**, напр.: Критичний, Високий, Середній, Низький)
- Дедлайн (дата виконання)
- Позначку виконання (IsCompleted)
- **Простроченість** — *обчислюване поле*:  
  `IsOverdue = !IsCompleted && DueDate < Today`

---

## Структура рішення
Рішення складається з 4 проєктів (мінімально необхідна структура):

- `TaskManager.Data`  
  Моделі **для зберігання** даних (ProjectData, TaskData) + enums

- `TaskManager.AppModels` (або `TaskManager.Presentation`)  
  Класи, що відповідають за **створення / відображення / редагування** (ProjectModel, TaskModel)

- `TaskManager.Services`  
  - `FakeStorage` (штучне сховище з початковими даними)  
  - `StorageService` (доступ до даних, завантаження, перетворення у моделі для UI)

- `TaskManager.ConsoleApp`  
  Консольний інтерфейс: меню, список проєктів, перегляд деталей, список завдань.

---

## Логіка консольного застосунку
Після запуску застосунок дозволяє:

1. **Завантажити та показати список проєктів**
   - виводиться коротка інформація (Id, Назва, Тип, Прогрес)

2. **Вибрати проєкт**
   - показати деталі проєкту (опис, прогрес)
   - завантажити завдання цього проєкту (якщо ще не завантажувались)
   - показати список завдань (назва, пріоритет, дедлайн, виконано/прострочено)

3. **Вибрати завдання**
   - показати повну інформацію (опис/характеристики, дедлайн, статус)

4. **Повернутись до списку проєктів** і повторити вибір

5. **Завершити роботу застосунку** штатно
